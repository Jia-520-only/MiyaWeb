## 概述

Web 应用安全是网络安全的重要组成部分。本文将深入分析常见的 Web 漏洞及其防护方案。

## XSS 跨站脚本攻击

### 什么是 XSS

XSS（Cross-Site Scripting）攻击者通过在网页中注入恶意脚本，当用户浏览该页面时，恶意脚本会在用户的浏览器中执行。

### XSS 的类型

1. **反射型 XSS**：恶意脚本通过 URL 参数注入
2. **存储型 XSS**：恶意脚本存储在服务器数据库中
3. **DOM 型 XSS**：恶意脚本通过 DOM 操作注入

### 防护措施

```javascript
// 输入验证和过滤
function sanitizeInput(input) {
  return input
    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;")
    .replace(/'/g, "&#x27;");
}

// 使用 Content Security Policy
// 在 HTTP 头中添加
Content-Security-Policy: default-src 'self'
```

## CSRF 跨站请求伪造

### 什么是 CSRF

CSRF（Cross-Site Request Forgery）攻击者诱骗用户在已认证的网站上执行非预期的操作。

### 防护措施

```javascript
// 生成 CSRF Token
const crypto = require('crypto');
function generateCSRFToken() {
  return crypto.randomBytes(32).toString('hex');
}

// 验证 CSRF Token
function validateCSRFToken(token, sessionToken) {
  return token === sessionToken;
}

// 在表单中添加 Token
app.use(express.csrf());
```

## SQL 注入

### 什么是 SQL 注入

SQL 注入是攻击者通过在输入字段中注入 SQL 代码，操纵后端数据库。

### 防护措施

```python
# 使用参数化查询（Python）
# ❌ 不安全的做法
query = f"SELECT * FROM users WHERE username = '{username}'"

# ✅ 安全的做法
query = "SELECT * FROM users WHERE username = ?"
cursor.execute(query, (username,))

# 使用 ORM
from sqlalchemy import text
query = text("SELECT * FROM users WHERE username = :username")
result = session.execute(query, {"username": username})
```

## 文件上传漏洞

### 常见风险

- 上传恶意文件（WebShell）
- 路径遍历攻击
- 文件类型绕过

### 防护措施

```python
import os

ALLOWED_EXTENSIONS = {'png', 'jpg', 'jpeg', 'gif'}

def allowed_file(filename):
  return '.' in filename and \
         filename.rsplit('.', 1)[1].lower() in ALLOWED_EXTENSIONS

def generate_unique_filename(filename):
  ext = filename.rsplit('.', 1)[1].lower()
  unique_name = uuid.uuid4().hex
  return f"{unique_name}.{ext}"

# 验证文件内容
def validate_file_type(file_stream):
  file_stream.seek(0)
  header = file_stream.read(4)
  # 验证文件头
  if header.startswith(b'\x89PNG'):
    return 'image/png'
  elif header.startswith(b'\xff\xd8'):
    return 'image/jpeg'
  return None
```

## 会话管理安全

### 安全的会话配置

```javascript
// Express.js 示例
app.use(session({
  secret: 'your-secret-key',
  resave: false,
  saveUninitialized: false,
  cookie: {
    secure: true,      // 仅 HTTPS
    httpOnly: true,    // 禁止 JavaScript 访问
    sameSite: 'strict',
    maxAge: 3600000    // 1 小时过期
  },
  name: 'sessionId'   // 自定义 cookie 名称
}));
```

### 会话固定防护

```javascript
app.use((req, res, next) => {
  if (req.session && !req.sessionID) {
    req.session.regenerate((err) => {
      if (err) return next(err);
      next();
    });
  } else {
    next();
  }
});
```

## 密码安全

### 密码哈希

```python
import bcrypt

# 哈希密码
def hash_password(password):
  salt = bcrypt.gensalt()
  return bcrypt.hashpw(password.encode('utf-8'), salt)

# 验证密码
def verify_password(password, hashed):
  return bcrypt.checkpw(password.encode('utf-8'), hashed)

# 使用示例
hashed = hash_password("myPassword123")
if verify_password("myPassword123", hashed):
  print("密码正确")
```

### 密码策略

```javascript
// 密码强度验证
function validatePassword(password) {
  const minLength = 8;
  const hasUpperCase = /[A-Z]/.test(password);
  const hasLowerCase = /[a-z]/.test(password);
  const hasNumbers = /\d/.test(password);
  const hasSpecialChar = /[!@#$%^&*(),.?":{}|<>]/.test(password);

  return password.length >= minLength &&
         hasUpperCase &&
         hasLowerCase &&
         hasNumbers &&
         hasSpecialChar;
}
```

## 总结

Web 安全是一个持续的过程，需要从多个层面进行防护：

1. **输入验证**：验证和过滤所有用户输入
2. **输出编码**：对输出到页面的内容进行编码
3. **身份认证**：使用强密码和多因素认证
4. **会话管理**：安全地管理用户会话
5. **HTTPS**：强制使用加密连接
6. **日志监控**：记录和分析安全事件

**安全不是一次性的工作，而是需要持续关注和改进的过程。**
