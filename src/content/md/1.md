## 前言

在这个数字化时代，服务器安全至关重要。本文将从零开始，详细讲解如何构建一个安全的 Linux 服务器环境。

## SSH 安全配置

SSH（Secure Shell）是远程管理 Linux 服务器的首要工具，因此确保其安全性非常重要。

### 修改默认端口

SSH 默认使用 22 端口，这是暴力破解攻击的首要目标。修改默认端口可以有效减少自动化攻击。

```bash
# 编辑 SSH 配置文件
sudo nano /etc/ssh/sshd_config

# 修改端口
Port 22222

# 重启 SSH 服务
sudo systemctl restart sshd
```

### 禁用 root 登录

禁止 root 用户直接登录可以大大提高安全性，建议使用普通用户 + sudo 的方式。

```bash
# 在 sshd_config 中添加或修改
PermitRootLogin no

# 重启服务
sudo systemctl restart sshd
```

### 使用密钥认证

密钥认证比密码认证更安全，建议禁用密码登录。

```bash
# 生成密钥对（在本地）
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

# 将公钥复制到服务器
ssh-copy-id -i ~/.ssh/id_rsa.pub user@your_server

# 在服务器上禁用密码登录
PasswordAuthentication no
```

## 防火墙设置

### UFW 基础配置

UFW（Uncomplicated Firewall）是 Ubuntu 上简单易用的防火墙工具。

```bash
# 安装 UFW
sudo apt update
sudo apt install ufw

# 设置默认策略
sudo ufw default deny incoming
sudo ufw default allow outgoing

# 允许 SSH（使用修改后的端口）
sudo ufw allow 22222/tcp

# 允许 HTTP/HTTPS
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 启用防火墙
sudo ufw enable
```

### iptables 高级规则

对于需要更精细控制的场景，可以使用 iptables。

```bash
# 创建自定义链
sudo iptables -N INCOMING

# 允许已建立的连接
sudo iptables -A INCOMING -m state --state ESTABLISHED,RELATED -j ACCEPT

# 限制每个 IP 的连接数
sudo iptables -A INCOMING -p tcp --dport 80 -m connlimit --connlimit-above 20 -j DROP
```

## 用户权限管理

### 创建普通用户

```bash
# 创建新用户
sudo adduser username

# 添加到 sudo 组
sudo usermod -aG sudo username
```

### 配置 sudo 权限

编辑 sudo 配置文件，可以精细控制用户权限。

```bash
sudo visudo

# 示例：允许特定用户执行特定命令
username ALL=(ALL) NOPASSWD:/usr/bin/apt
```

## 系统更新与监控

### 自动更新配置

```bash
# 安装自动更新工具
sudo apt install unattended-upgrades

# 配置自动更新
sudo dpkg-reconfigure --priority=low unattended-upgrades
```

### 日志监控

```bash
# 查看登录日志
sudo lastlog

# 查看失败登录尝试
sudo grep "Failed password" /var/log/auth.log

# 实时监控日志
sudo tail -f /var/log/auth.log
```

## 其他安全建议

### 安装入侵检测系统

Fail2ban 可以自动封禁暴力破解的 IP 地址。

```bash
sudo apt install fail2ban
sudo systemctl enable fail2ban
sudo systemctl start fail2ban
```

### 定期备份

```bash
# 使用 rsync 备份重要文件
rsync -avz /home/user/ /backup/user/

# 配置定时任务
crontab -e
# 每天凌晨 2 点备份
0 2 * * * rsync -avz /home/user/ /backup/user/
```

## 总结

通过以上步骤，我们已经构建了一个相对安全的 Linux 服务器环境。但安全是一个持续的过程，需要定期审查和更新配置。

记住：**安全没有终点，只有不断进步。**
